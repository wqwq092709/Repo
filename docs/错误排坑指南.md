# 苍穹外卖项目——错误排坑指南

## 环境信息

| 项目 | 版本 |
|------|------|
| Spring Boot | 2.7.3 |
| Knife4j (内置 Springfox) | 3.0.2 / 3.0.0 |
| Maven | 3.9.x |
| Java | 17 |

启动命令：`mvn package -DskipTests && java -jar sky-server/target/sky-server-1.0-SNAPSHOT.jar`

## 错误 1：Maven 报 Non-parseable settings

### 报错信息

```
[FATAL] Non-parseable settings /Users/xxx/maven-3.9/apache-maven-3.9.14/conf/settings.xml:
end tag name </settings> must match start tag name <mirrors> from line 162
```

### 原因

`settings.xml` 中 `<mirrors>` 标签未闭合——闭合标签 `</mirrors>` 被错误地放在了 XML 注释 `<!-- ... -->` 内部，解析器识别不到。

```xml
<!-- 错误示例 -->
<mirrors>
    <mirror>
        <id>aliyunmaven</id>
        ...
    </mirror>

<!--  </mirrors> 被注释掉了！ -->
<!-- <mirror>
        ...
    </mirror>
  </mirrors> -->
```

### 修复

将 `</mirrors>` 移出注释，放在最后一个 `<mirror>` 与 `<!--配置私服-->` 注释之间：

```xml
<mirrors>
    <mirror>
        <id>aliyunmaven</id>
        ...
    </mirror>
  </mirrors>   <!-- ← 必须是有效的 XML 标签，不能放在注释里 -->

<!--配置私服的访问路径-->
<!-- <mirror> ... </mirror> -->
```

**验证：** `mvn -version` 不再报错即可。

## 错误 2：Springfox NPE — documentationPluginsBootstrapper 启动失败

### 报错信息

```
Error starting ApplicationContext. To display the conditions report re-run your application with 'debug' enabled.

org.springframework.context.ApplicationContextException:
Failed to start bean 'documentationPluginsBootstrapper';
nested exception is java.lang.NullPointerException:
Cannot invoke "org.springframework.web.servlet.mvc.condition.PatternsRequestCondition.getPatterns()"
because "this.condition" is null
    at springfox.documentation.spring.web.WebMvcPatternsRequestConditionWrapper.getPatterns(...)
    at springfox.documentation.RequestHandler.sortedPaths(...)
```

### 原因

Spring Boot 2.6+ 默认路径匹配策略从 `AntPathMatcher`（返回 `PatternsRequestCondition`）改为 `PathPatternParser`（返回 `PathPatternsRequestCondition`）。Knife4j 3.0.2 内置的 **Springfox 3.0.0** 只认识前者，遇到 `PathPatternsRequestCondition` 时 `condition` 字段为 null，导致 NPE。

> ⚠️ **特别注意：** 如果项目中 `WebMvcConfiguration` 继承了 `WebMvcConfigurationSupport`，Spring Boot 的 WebMvc 自动配置会被禁用，`application.yml` 中的 `spring.mvc.pathmatch.matching-strategy=ant_path_matcher` **不会生效**。

### 修复（推荐方案：BeanPostProcessor）

新增 `SpringfoxBeanPostProcessor.java`：

```
sky-server/src/main/java/com/sky/config/SpringfoxBeanPostProcessor.java
```

内容：实现 `BeanPostProcessor`，在 `WebMvcRequestHandlerProvider` 初始化后，反射获取 `handlerMappings` 字段，过滤掉 `PatternParser != null` 的 mapping（即使用了新策略的 mapping），只保留使用 `AntPathMatcher` 的 mapping。

**为什么这个方案最好：**
- 不需要改 `application.yml`
- 不需要改 `WebMvcConfiguration`
- 兼容继承了 `WebMvcConfigurationSupport` 的场景
- 即使以后有其他配置干扰路径匹配策略，这个兜底也能保证不炸

### 备选方案（如果还没继承 WebMvcConfigurationSupport）

在 `application.yml` 中添加：
```yaml
spring:
  mvc:
    pathmatch:
      matching-strategy: ant_path_matcher
```

**但这个方案有前提：** 不能继承 `WebMvcConfigurationSupport`（改用 `WebMvcConfigurer` 接口），否则配置不生效。

### 验证

启动后看到 `Started SkyApplication in X.XXX seconds` 且没有上述 NPE 报错即为修复成功。

## 排查技巧

1. **看完整堆栈，找 Caused by —** 根因通常在最底部的 `Caused by` 行。
2. **Springfox 报 NPE 就是版本兼容问题 —** Springfox 3.0.0 已停更，与 Spring Boot 2.6+/Spring MVC 5.3+ 的兼容性靠 workaround 兜底。
3. **Maven 报 XML parse error —** 先检查 `settings.xml` 和 `pom.xml` 的标签是否成对闭合，尤其是被注释包裹的标签。
